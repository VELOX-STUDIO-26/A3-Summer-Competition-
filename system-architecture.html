System Architecture
Graph-Based Component Topology
▼
📐 Architecture Layers
Client Layer (Web Only)
Gateway Layer (API/WebSocket/Rate Limit)
Core Services (Orchestrator/Planner/Tutor/NLP/Adaptation)
AI Agent Layer (9 Agents)
External Services (LLM/ASR/TTS/Judge0)
Data Layer (PostgreSQL/Redis)
🔄 Data Flow
Request Flow: Client → Gateway → Services | Services → Agents → OpenRouter (LLM)
Direct Service Usage: Tutor Engine → OpenRouter+Weaviate | NLP → OpenRouter+iFlytek | Adaptation → OpenRouter
Orchestrator Pattern: Only Orchestrator dispatches to 5 agents (Content, Quiz, MindMap, Media, Code)
Persistence: LLM results → PostgreSQL | TTS audio → Redis cache | Vector data → Weaviate
Code Execution: Architect → Judge0 | Grader → Judge0
